# QuickLabel

Локальное web-приложение для **быстрой разметки датасетов** с ускорением через
**SAM 2** (интерактивно) и **SAM 3** (авторазметка по текстовому запросу).
Изначально создано на основе **VisoLabel**, но теперь **полностью автономно**:
содержит свою копию SAM-бэкенда (`QuickLabel/ml_backend`) и веса моделей
(`QuickLabel/models/sam2.1_hiera_large.pt`, `QuickLabel/models/sam3.pt`).
Папка `VisoLabel` больше не нужна.

## Почему web-приложение

GUI исходного VisoLabel скомпилирован в `.exe` (исходников нет), а вот исходники
SAM-бэкенда доступны. Локальное web-приложение проще разрабатывать и
поддерживать: backend на Python переиспользует готовый `SAMService`, а UI — это
обычные HTML/CSS/JS без шага сборки.

## Установка (один раз)

Нужен **Python 3.13**. Из папки `QuickLabel` запустите скрипт установки —
он создаст `.venv` и поставит всё (включая встроенные wheel'ы SAM из `wheels/`):

```powershell
.\setup.ps1            # сборка под NVIDIA GPU (CUDA 12.4)
.\setup.ps1 -CpuOnly   # без GPU (SAM 3 будет медленным)
```

> `triton` не требуется — `ml_backend/sam3_compat.py` даёт OpenCV-фолбэк.
> numpy 2.x работает, несмотря на консервативный пин `<2` у sam3.
> `sam3` нет на PyPI, поэтому он ставится из `wheels/sam3-…whl` (`--no-deps`).

## Запуск

**Двойной клик по `QuickLabel.exe`** в папке проекта. Появится окно консоли с
логом сервера, через ~2 секунды откроется браузер на `http://127.0.0.1:8765`.
Чтобы остановить — закройте окно.

Альтернативно: `.\run.ps1` или `.\run.bat` (то же самое, без .exe). При желании
.exe можно пересобрать: `.\build_exe.ps1` (нужен `pyinstaller` в venv).

`QuickLabel.exe` — это **лёгкий лаунчер (~7 МБ)**: он не содержит Python и
зависимости, а просто находит ваш venv и запускает сервер. Поэтому в той же
папке (или в `annotation_SAM/`) должен быть `.venv`, созданный через `setup.ps1`.

> Первый вызов SAM 2/SAM 3 загружает модель в память (SAM 2 ~9с, SAM 3 ~13с
> на GPU). Дальнейшие вызовы быстрые: SAM 3 ~1с на изображение.

> **Производительность на картах до Ampere (RTX 20xx / Turing):** SAM 3
> использует `float16` autocast вместо `bfloat16`. У Turing нет тензорных ядер
> для bf16, из-за чего энкодер работал ~60с на фото; с fp16 — ~1с. Выбор dtype
> автоматический по compute capability GPU (см. `ml_backend/sam_service.py`).

## Перенос на другой компьютер

Приложение спроектировано переносимым. Что нужно сделать:

1. **Скопировать папку `QuickLabel`** целиком — в ней уже есть код, веб-интерфейс,
   SAM-бэкенд (`ml_backend/`), веса моделей (`models/*.pt`, ~4.3 ГБ) и wheel'ы SAM
   (`wheels/`). Папка `VisoLabel` не нужна.
2. **НЕ копировать `.venv`** — виртуальные окружения непереносимы (внутри абсолютные
   пути и бинарники под конкретную ОС). На новом ПК окружение пересоздаётся.
3. На новом ПК: установить **Python 3.13**, затем из папки `QuickLabel` выполнить
   `.\setup.ps1` (или `.\setup.ps1 -CpuOnly`). Скрипт сам поставит torch, sam2/sam3
   из `wheels/` и остальное.
4. Запустить `.\run.ps1` — он сам найдёт `.venv` (внутри `QuickLabel` или рядом).

Требования на целевом ПК для GPU-ускорения: **видеокарта NVIDIA** и свежие драйверы
(CUDA подтягивается внутри пакета torch, отдельная установка CUDA Toolkit не нужна).
Без NVIDIA приложение работает на CPU (SAM 3 — медленно, минуты на фото).

Папку `projects/` можно переносить вместе с приложением, если нужны и сами данные
разметки; пути к изображениям, импортированным «по папке», на новом ПК должны
совпадать (либо загружайте изображения кнопкой «Загрузить файлы» — тогда они
копируются внутрь проекта и переносятся вместе с ним).

> Чек-лист переносимости: `QuickLabel/` (код + `QuickLabel.exe`) +
> `models/*.pt` (веса) + `wheels/` (SAM) + Python 3.13 на целевом ПК +
> `setup.ps1`. Всё. После `setup.ps1` запуск — двойной клик по `QuickLabel.exe`.

## Возможности

- **Проекты**: создание/открытие/**удаление** (кнопка «🗑 Удалить» рядом с выбором
  проекта), всё хранится в `projects/<slug>/project.json`.
- **Изображения**: импорт локальной папки (без копирования) или загрузка файлов;
  удаление отдельного изображения кнопкой «✕» в списке слева.
- **Секундомер SAM 3**: время разметки фото логируется в консоль сервера
  (`[SAM3] …`) и показывается в панели «Авторазметка (SAM 3)» (⏱). Только SAM 3.
- **Классы**: создание, выбор, цвет, удаление.
- **Ручная разметка**: «Рамка» (прямоугольник) и «Полигон» (многоугольник —
  клики по точкам, замыкание двойным кликом или Enter, отмена — Esc). Полигон
  нужен для объектов под углом (например, смотровое стекло).
- **Статичные рамки (ROI на все кадры)**: галочка у каждой рамки в списке
  аннотаций. Включённая рамка хранится на уровне проекта (`static_rois`) в
  нормализованных координатах (0..1 от размеров изображения) и применяется ко
  ВСЕМ кадрам — задаёте ROI один раз, измерение идёт внутри тех же областей на
  каждом кадре. **Работает на изображениях разного размера** (координаты
  масштабируются по размеру каждого кадра). На холсте — пунктир + 📌.
- **Исключение «всем, кроме этого кадра»**: рядом со статичной рамкой вторая
  галочка «применять на этом кадре». Снимите её, чтобы исключить эту рамку
  только на текущем кадре (хранится в `roi.exceptions: [image_id…]`). Полезно,
  когда на отдельном кадре камера чуть сместилась.
- **Палитра цвета класса**: клик по цветному образцу у класса открывает выбор
  цвета; цвет применяется к контуру/заливке всех рамок этого класса.
- **Горячие клавиши**: `1` — Рамка, `2` — Полигон, `3` — SAM-точка, `4` — SAM-рамка,
  `←`/`→` — предыдущий/следующий кадр, `+`/`−`/`0` — зум/вписать,
  `Enter`/`Esc` — принять/отменить, `Delete` — удалить выбранную рамку.
- **SAM 2**: «SAM-точка» (клик по объекту, Shift = отрицательная точка) и
  «SAM-рамка» (обвести объект) → предлагается маска/полигон для подтверждения.
- **SAM 3 по классу или запросу**: при выборе класса запрос подставляется
  автоматически (= имя класса), можно переопределить любым текстом. Результат
  записывается в выбранный класс. Так связка работает в обе стороны: разметили
  объект вручную через SAM 2 → выбрали класс → SAM 3 находит «такой же» на
  остальных фото; либо ищете сразу по текстовому запросу.
- **Распространение на изображения**: панель «Авторазметка (SAM 3)» — выбор
  класса, запроса и области («все» / «следующие за текущим»). SAM 3 прогоняется
  по изображениям и создаёт предложения (пунктир), которые подтверждаются/правятся.
- **Прогресс и отмена**: все запросы SAM выполняются как фоновые задачи с
  прогресс-баром («кодирование», «поиск», «изображение X из N») и кнопкой
  «Отменить». Отмена распространения останавливает обработку между изображениями.
- **Память GPU**: на 8 ГБ карте одновременно держится только одна модель
  (SAM 2 ↔ SAM 3 переключаются автоматически); при нехватке VRAM выдаётся
  понятное сообщение вместо зависания.
- **Экспорт (кнопка «Экспорт ▾» — выпадающий список)**:
  - **YOLO** — `images/{train,val}` + `labels/{train,val}` + `data.yaml`,
    detection (рамки) или segmentation (полигоны).
  - **RF-DETR — COCO JSON (segmentation)** — Roboflow-структура `train/` и
    `valid/` с `_annotations.coco.json`; категории с placeholder id 0, классы с
    id 1; каждая аннотация содержит `bbox` + `segmentation` (подходит и для
    RF-DETR detection, и для RF-DETR-Seg).
  - Оба: авторазделение 90/10, статичные ROI на каждый кадр, аугментация.
- **Аугментация**: повороты (90/180/270° или любые) с точным пересчётом
  аннотаций — добавляются в train-часть экспорта.

## Архитектура

```
QuickLabel/
  backend/
    config.py        # пути, подключение ml_backend, настройки сервера
    store.py         # проекты/классы/изображения/аннотации на диске (JSON)
    sam_runtime.py   # in-process обёртка над ml_backend SAMService (SAM 2/3)
    export_common.py # общий конвейер: split + статичные ROI + аугментация
    yolo_export.py   # экспорт YOLO (Ultralytics)
    coco_export.py   # экспорт COCO JSON для RF-DETR (Roboflow-структура)
    server.py        # FastAPI: REST API + раздача web/
  ml_backend/        # копия SAM-сервиса (sam_service, protocol, sam3_compat)
  models/            # веса: sam2.1_hiera_large.pt, sam3.pt
  web/               # index.html, app.js, style.css (vanilla, без сборки)
  run.ps1 / run.bat  # запуск через проектный venv
  projects/          # данные проектов
```

### Поток авторазметки (SAM 3)

1. Пользователь вводит текстовый запрос (напр. `car`) и жмёт **SAM 3 ►** —
   получает предложения на текущем изображении, подтверждает их.
2. Жмёт **Распространить на следующие** — backend (`/sam3/propagate`) прогоняет
   запрос по всем последующим изображениям и сохраняет предложения
   (`status: "suggested"`), не затирая уже подтверждённую работу.
3. Пользователь листает изображения, нажимает **Принять все предложения** или
   правит/удаляет отдельные аннотации.

## Настройка путей

Через переменные окружения (необязательно):

- `VISOLABEL_DIR` — папка VisoLabel (по умолчанию соседняя `../VisoLabel`).
- `QUICKLABEL_PROJECTS` — где хранить проекты.
- `QUICKLABEL_HOST` / `QUICKLABEL_PORT` — адрес сервера.
- `ML_BACKEND_SAM2_PATH` / `ML_BACKEND_SAM3_PATH` — явные пути к весам.

## Формат YOLO

- **detection**: `class cx cy w h` (нормировано 0..1).
- **segmentation**: `class x1 y1 x2 y2 …` (нормированный полигон).
- `data.yaml`: `path`, `train`, `val`, `nc`, `names` (id ремаппятся в 0..N-1).
- Деление train/val — на уровне исходных изображений (аугментации идут только в
  train), чтобы избежать утечки данных в валидацию.
- В экспорт попадают подтверждённые аннотации; неподтверждённые предложения —
  только при включённой опции.
